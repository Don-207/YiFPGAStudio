# 安全策略

## 支持范围

YiFPGA Studio 尚处于 1.0.0 候选阶段。安全修复优先应用于 `main`，正式版本发布后，
本文件将列出仍接受安全更新的版本系列。

## 报告安全问题

请不要为尚未修复的漏洞创建公开 Issue。优先使用本仓库 GitHub 页面中的
**Security → Report a vulnerability** 发起私密报告。

报告中请尽量包含：

- 受影响的提交、版本、模块或脚本；
- 可复现步骤和最小测试用例；
- 可能造成的影响；
- 已确认的器件、工具版本和运行环境；
- 建议的缓解方法（如有）。

不要在报告中附带私钥、访问令牌、未脱敏的硬件标识或其他第三方机密信息。

维护者确认问题后，会通过私密报告线程协调复现、修复和披露时间。在修复公开前，
请避免披露可直接利用的技术细节。

## 硬件安全边界

仓库中的示例寄存器窗口、JTAG Bridge 和板级 Demo 面向开发与调试环境，不应直接视为
生产系统的安全边界。将其用于实际产品前，应独立评估访问控制、调试口禁用策略、
输入校验、权限隔离和升级恢复机制。
